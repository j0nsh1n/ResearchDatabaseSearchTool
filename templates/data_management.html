Getting started

Three quick steps. Preparing papers for search runs itself after the fetch.

  1. ○ Pick one or more topics below
  2. ○ Type a research question in Fetch
  3. ○ Press Fetch Articles, then Prepare Papers when ready

Prefer a dry run with no APIs? (~20 education-themed demos; replaces an empty collection)

1. Select Topics

Choose the academic areas you care about - we use them to recommend sources.

How topics are used

Topics drive two things: which databases get recommended below, and the coverage map after a fetch (are you missing sources teachers often expect for this topic?). Your collection stays private to this account.

2. Choose Sources

Select topics above to see recommended sources, or choose manually below.

About the sources

Each source is a free academic database. Some need free API keys (CORE, NASA ADS); without a key they simply skip - the rest still run.

3. Fetch Articles

Searches every selected source at once and saves the papers to your private collection. Preparing for search is a separate step (button below) so you control the model and timing.

How fetching works

The query runs against every selected source in parallel as a background job (progress bar; Cancel stops after the current source). Saves title, abstract, authors, year, and journal. Records without an abstract are skipped (later steps need text). Some sources may fail (rate limits, API rules) while others succeed — that is normal; check the per-source list. Your last query, sources, and email are remembered in this browser.

Advanced fetch options

Coverage map

A quick health check after a fetch: papers per source, and which recommended sources are still empty.

4. Prepare Papers for Search

Run this after a fetch when you are ready. The analysis model follows your selected topics (or your Advanced override). Nothing here starts until you press the button.

What preparing does (embeddings)

Preparing creates embeddings: each abstract becomes a numerical "meaning fingerprint" that powers Search (hybrid ranking, seed, and more-like-starred), Clusters, and Duplicate detection. It runs as a background job with a progress bar (you can leave the page open) and uses your GPU when available. The model is chosen from your topics (biomedical → pubmedbert, hard science → specter, otherwise general) unless you pick one under Advanced. Only prepare new papers turns on for Add to collection and off for Replace. Switching models reprocesses everything so dimensions stay consistent.
