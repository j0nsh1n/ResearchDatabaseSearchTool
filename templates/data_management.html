Data Management

Step 1 of 4. Select topics, fetch papers, then create embeddings. Fetch and embed run as background jobs (progress bars; safe if you leave the tab open). When you’re done → Clusters to group and screen topics.

1. Select Topics

Choose the academic areas you care about. We use this to recommend databases and to check your coverage map (are you missing sources teachers often expect for this topic?). Your collection stays private to this account.

2. Choose Sources

Select topics above to see recommended sources, or choose manually below. Some sources need free API keys (CORE, NASA ADS); without a key they simply skip.

Coverage map

A quick health check: how many papers you already have per database, and which recommended sources are still empty. Re-run after a fetch finishes (background jobs update the bars when you refresh).

3. Fetch Articles

Runs your query against every selected source in parallel as a background job (progress bar; if one fetch is already running you will be told to wait). Saves title, abstract, authors, year, and journal into your private database. Records without an abstract are skipped (later steps need text). Your last query, sources, and email are remembered in this browser.

4. Create Embeddings

Embeddings turn each abstract into a numerical “meaning fingerprint.” They power Search (hybrid ranking, seed, and more-like-starred), Clusters, and Duplicate detection - run this after every fetch. The run is a background job with a progress bar (you can leave the page open). Pick general for speed or mixed topics; pubmedbert / biosentbert for medical corpora; specter for scientific papers. Only embed new papers turns on automatically when you choose Add to collection (so existing vectors are kept); for a full Replace fetch it stays off so everything is embedded. Switching models re-embeds everything. Uses your GPU when available (including ROCm on supported Linux setups).
