Duplicate Detection

Finds near-identical papers across your sources so you can keep one copy of each. Removed copies are screened out, never deleted.

How detection and auto-resolve work

Detection needs prepared papers (Data Management, step 4). Match strictness: higher values find only near-identical articles, lower values find more loosely related ones. Auto-Resolve All keeps the most complete abstract in each group; if lengths are similar it prefers higher-quality sources (PubMed > Europe PMC > CrossRef > OpenAlex > …). Redundant copies are screened out (hidden from Search, not deleted) and the screening report counts update. You can restore anything from its cluster on the Clusters page, or use per-group Keep this to choose the winner by hand.

Higher finds only near-identical papers; lower catches looser matches.

Screening report

Show-your-work counts for hand-ins (not a citation list). Use this when a teacher asks how many papers you collected, excluded, and kept.

What this file is for (vs Search exports)

This is a plain-text accounting report (PRISMA-style numbers): papers collected, removed as duplicates, excluded by cluster triage, excluded manually, included set, stars, prepared papers, clusters, and papers-by-year. The on-screen view and Download (.txt) use the same text - good to attach to a reflection or lab report.

It is not a bibliography. For citation files, use Search → Export your library (RIS for Zotero).
